02 — Stack
Technology choices

TypeScript-only stack. Hono.js backend (unchanged). Next.js 15 App Router frontend — RSC reads, Server Actions for mutations, ISR for public portal. Zustand for auth, TanStack Query for WebSocket-invalidated data only.

Frontend
Next.js 15
App Router. RSC for server reads, Server Actions for mutations, ISR for public portal, SSG for static pages. Edge middleware for auth guard. No Vite, no React Router.
Global State
Zustand
One store, ~50 lines — auth token + user only. TanStack Store removed. RSC eliminates the need for client-side server state management on dashboard pages.
Styling
Tailwind CSS
Utility-first CSS. Fast prototyping, dark mode built-in, responsive by default.
Server State
TanStack Query v5
Used only for WebSocket-invalidated client data (comments, live notifications). RSC handles all other data fetching server-side.
Backend
Hono.js + Node.js
TypeScript-native web framework. 5× faster than Express, built-in Zod validation via zValidator, edge-compatible.
Authentication
JWT + bcrypt
Stateless auth with access + refresh token pattern. Passwords hashed with bcrypt (12 rounds).
File Storage
ImageKit
Unified media platform. SDK for Node.js backend uploads + imagekitio-react for optimised delivery with real-time transformations and LQIP.
Real-time
Hono WebSocket
Native WebSocket via @hono/node-ws. Custom room manager replaces Socket.io — lighter, no dependency on Native WebSocket + custom wsClient singleton.
Database
MongoDB Atlas
Managed cloud MongoDB. Flexible documents for updates, nested comments, and file metadata.
ODM
Mongoose
Schema validation, virtuals, middleware hooks (pre-save), and population for relational queries.
Email
AWS SES
@aws-sdk/client-ses for transactional email. Production-grade deliverability, domain verification, bounce/complaint handling via SNS.
Deployment
Vercel + Railway
React 19 on Vercel (edge CDN). Hono.js on Railway/Fly.io. MongoDB Atlas managed service.

📁 Project Model
  • Project schema: title, description, status, slug, clientEmail, ownerId
  • CRUD routes: create, read, update, delete project
  • Unique shareable slug generation (nanoid)
  • Owner-only access middleware
  • GET /api/projects/:slug/public — unauthenticated client view
Frontend Tasks
⚛️ React Setup
  • Next.js 15 App Router + TypeScript strict mode — no Vite, no React Router
  • File-based routing — folder = route segment, zero config
  • Tailwind CSS configuration
  • Zustand: authStore — token + user only (~50 lines, replaces TanStack Store)
  • Native fetch() wrapper (lib/api.ts) — server-side only, reads JWT from httpOnly cookie

  • accessToken in httpOnly cookie — middleware reads it at edge, no client JS needed for auth
  • Token refresh handled server-side in Server Actions — no client-side interceptor needed
🗂️ Dashboard Shell
  • Sidebar nav layout component
  • Projects list page — fetch + display projects
  • Create project modal with form
  • Project status pill component
  • Empty state illustrations

04 — App Flow
End-to-end application flow

How data and users move through the entire system — from registration to client portal view.

Provider journey
1. Register / Login
Email + password → JWT access & refresh tokens stored
→
2. Dashboard
View all projects, status overview, notifications
→
3. Create Project
Title, client email, description → unique slug generated
→
4. Share Portal Link
Copy link or send via email to client directly
5. Post Update
Text + optional files + milestone flag → saved to DB
→
6. File Upload
Multer → Cloudinary → URL stored in update document
→
7. Email Trigger
Client notified of new update via Nodemailer/Resend
→
8. Track Activity
See when client viewed, commented, approved
↕ shared project data ↕
Client journey (no login required)
1. Receive Link
Email or shared URL: app.com/p/[slug]
→
2. Portal Page
Project title, status, progress — public route, no auth
→
3. View Updates
Chronological timeline, file previews, milestones
→
4. Download Files
Cloudinary-signed URLs, direct browser download
5. Leave Comment
Name + email form, comment saved, owner notified
→
6. Approve Milestone
One-click approve or request revision on milestone cards
→
7. Real-time Echo
Socket.io emits to owner: client_viewed, approved
→
8. Owner Notified
Toast + bell update + optional email to freelancer
